# 구성원 등록(인사정보) 요구사항 정의서

## 1) 목적 / 범위
- **목적**: 관리자(ROLE_ADMIN)가 구성원 인사정보를 등록하고, 고용 형태에 따라 보험/세금/계약 종료일 관리가 달라지는 정책을 반영한다.
- **현재 범위(MVP)**:
  - 구성원 등록: `/admin/employees/new`
  - 구성원 목록/필터: `/admin/employees?type=...`
  - 계정 발급(초기 비밀번호 1회 표시): 정규직만 가능
  - 삭제: 인사정보 삭제(미발급), 계정+인사정보 삭제(발급됨, 정규직만)
- **비범위(후속)**: 사용자 포털, 비밀번호 재발급/초기화, 최초 로그인 비밀번호 변경 강제, 계약 이력(1:N), 보험 4종 세부 관리, 급여/세금 계산 엔진 연동

## 2) 사용자 / 권한
- **관리자(ROLE_ADMIN)**:
  - 구성원 인사정보 등록/조회/삭제
  - 정규직 계정 발급 및 계정 포함 삭제
- **일반 사용자(ROLE_USER)**:
  - (현재 MVP에서는 사용자 홈만 존재) `/user`

## 3) 화면/동선(IA)
- **관리자**
  - 홈: `/admin/home`
  - 설정: `/admin/settings`
  - 구성원 목록: `/admin/employees`
  - 구성원 등록: `/admin/employees/new`
- **인증/리다이렉트**
  - `/`:
    - 미로그인 → `/login`
    - 관리자 → `/admin/home`
    - 일반 사용자 → `/user`

## 4) 데이터 모델(현재 엔티티 기준)

### 4.1 employees (인사정보)
- PK: `emp_no` (사번)
- 주요 컬럼
  - `emp_no` (String): 사번(숫자만)
  - `name` (String): 이름
  - `employment_type` (Enum String): `REGULAR|CONTRACT|FREELANCER`
  - `four_insured` (Boolean): 4대 보험 가입 여부
  - `tax_scheme` (Enum String): `SALARY_WITHHOLDING|BUSINESS_INCOME_3_3`
  - `contract_end_date` (LocalDate): 계약 종료일
  - `created_at` (LocalDateTime)
- 관계
  - `employees (1) <-> (0..1) users` (users.employee_no FK + UNIQUE)

### 4.2 users (계정)
- PK: `username` (사번을 그대로 사용)
- 주요 컬럼
  - `pwd` (BCrypt hash)
  - `role` (`ADMIN|USER`)
  - `enabled` (boolean)
  - `employee_no` (FK): employees.emp_no (관리자 계정은 null 가능)

## 5) 필드 정의(등록 화면 기준)

| 구분 | 필드 | 화면 입력 | 저장 컬럼 | 타입 | 필수 | 제약/검증 | 비고 |
|---|---|---|---|---|---|---|---|
| 기본 | 사번 | 텍스트(숫자) | employees.emp_no | String | Y | `^[0-9]+$`, PK/중복불가 | 사번=계정ID로 사용 |
| 기본 | 이름 | 텍스트 | employees.name | String | Y | 공백불가 |  |
| 고용| 고용형태 | Select | employees.employment_type | Enum | Y | REGULAR/CONTRACT/FREELANCER | 폼 토글 기준 |
| 고용 | 4대보험 | Select | employees.four_insured | Boolean | 조건부 | 정규/계약: 선택(기본 true), 프리: false 고정 | MVP 정책 |
| 고용 | 세금방식 | Hidden(자동) | employees.tax_scheme | Enum | Y | 정규/계약: SALARY_WITHHOLDING 고정, 프리: BUSINESS_INCOME_3_3 고정 | UI에서 자동 세팅 |
| 계약 | 계약종료일 | Date | employees.contract_end_date | LocalDate | 조건부 | 계약/프리: 필수, 정규: null | 만료 운영정책 연계 |

## 6) 업무 규칙(핵심 정책)

### 6.1 고용형태별 입력/저장 정책
- **정규직(REGULAR)**
  - `tax_scheme`: 근로소득(원천징수) 고정
  - `four_insured`: 입력값이 없으면 true로 기본
  - `contract_end_date`: null
- **계약직(CONTRACT)**
  - `contract_end_date`: 필수
  - `tax_scheme`: MVP에서는 근로소득(원천징수) 고정
  - `four_insured`: 입력값이 없으면 true로 기본
- **프리랜서(FREELANCER)**
  - `contract_end_date`: 필수
  - `tax_scheme`: MVP에서는 사업소득(3.3%) 고정
  - `four_insured`: false 고정(입력 불가)

### 6.2 계정 발급/삭제 정책
- **계정 발급**: 정규직만 가능
  - username = emp_no
  - 초기 비밀번호 랜덤 생성 → BCrypt 저장 → 화면에 1회 표시
- **삭제**
  - 미발급: 인사정보만 삭제 가능
  - 발급됨: “계정+인사정보 삭제”로 처리(FK 때문에 계정 먼저 삭제)
  - **계정 포함 삭제는 정규직만 가능**

## 7) 예외/에러 케이스(대표)
- **사번 중복**: `Employee already exists`
- **사번 형식 오류**: 숫자만 가능
- **계약직/프리랜서 계약종료일 누락**: 필수 오류
- **계정 발급 제한**: 정규직이 아니면 발급 불가
- **계정 중복(username exists)**: users에 동일 사번이 이미 존재하면 발급 불가
- **계정 삭제 제한**: 정규직이 아니면 계정 포함 삭제 불가

## 8) 보안/감사(권장)
- **권한 체크**: `/admin/**`는 ROLE_ADMIN만 접근(보안 설정)
- **비밀번호 정책(후속 권장)**:
  - 최초 로그인 시 변경 강제
  - 초기 비밀번호 만료/재발급 기능
  - 로그인 실패 횟수 제한/계정 잠금
- **감사 로그(후속 권장)**:
  - 누가/언제/어떤 사번을 등록/수정/삭제/발급했는지 기록

## 9) 운영 관점 체크리스트
- 사번 정책(숫자만/자리수/선행0/변경 가능성) 확정
- 고용형태 확장 가능성(인턴/파트/파견 등) 고려
- 계약 종료일 기반 자동 처리(알림/계정 비활성화) 필요 여부
- “삭제” 대신 “퇴사/비활성화(soft delete)”로 바꿀지 결정

## 10) 후속 확장 제안(우선순위 예시)
1. **비밀번호 재발급/초기화**(관리자 기능)
2. **최초 로그인 비밀번호 변경 강제**
3. 계약 이력 테이블(연장/재계약 트래킹)
4. 보험 4종 세부 체크(국민/건강/고용/산재)
5. 급여/세금 정산 모듈 연동(원천징수/3.3% 계산)
